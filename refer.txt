        <h1 id="第1期---新的开始">第1期 - 新的开始</h1>

<p>封面图拍摄于今天下午，气温25左右，属于是这段时间内最热的一天了，骑车到附近的一个小山包。</p>

<h2 id="技术栈">技术栈</h2>

<p>这套技术栈成本可以是0，唯一需要花钱的就是域名了，第一年一般10刀以内，往后会贵一些。</p>

<h3 id="工具">工具</h3>

<ul>
  <li>Trae：第一个月会员3刀，到期可以换个账号，里面的Claude-4-sonnet真的让人割舍不了。</li>
  <li>Google Gemini CLI：免费里面最好用的了。</li>
</ul>

<h3 id="核心框架">核心框架</h3>
<ul>
  <li>Jekyll 4.x ：静态网站生成器，基于 Ruby</li>
  <li>GitHub Pages ：通过 github-pages gem 集成，版本 ~228
    <h3 id="内容处理">内容处理</h3>
  </li>
  <li>Kramdown ：Markdown 解析器，支持 GitHub Flavored Markdown (GFM)</li>
  <li>Rouge ：代码语法高亮</li>
  <li>MathJax ：数学公式渲染</li>
  <li>Mermaid ：图表和流程图支持
    <h3 id="jekyll-插件">Jekyll 插件</h3>
  </li>
  <li>jekyll-feed ：RSS 订阅功能</li>
  <li>jekyll-sitemap ：自动生成站点地图</li>
  <li>jekyll-seo-tag ：SEO 优化标签
    <h3 id="前端技术">前端技术</h3>
  </li>
  <li>原生 JavaScript ：用于博客交互功能（分页、搜索、标签过滤等）</li>
  <li>CSS3 ：样式设计，包括渐变背景和响应式布局</li>
  <li>SCSS ：CSS 预处理器（通过 Jekyll 处理）
    <h3 id="字体和资源">字体和资源</h3>
  </li>
  <li>LXGW WenKai 字体 ：中文显示字体</li>
  <li>Font-display: swap ：优化字体加载性能
    <h3 id="部署平台">部署平台</h3>
  </li>
  <li>GitHub Pages ：主要部署平台，通过 GitHub Actions 自动部署</li>
  <li>Vercel ：备用部署平台，支持自定义域名 pingfan.me
    <h3 id="分析工具">分析工具</h3>
  </li>
  <li>Vercel Analytics ：网站访问统计
    <h3 id="性能优化">性能优化</h3>
  </li>
  <li>DNS 预解析和预连接（针对 CDN 资源）</li>
  <li>资源预加载（字体、CSS）</li>
  <li>响应式图片和布局</li>
</ul>

### 关于时间管理

这周在思考时间管理的问题。发现自己经常陷入"忙碌陷阱"——看起来很忙，但实际产出不高。

尝试了番茄工作法，效果还不错：
- 25分钟专注工作
- 5分钟休息
- 每4个番茄休息15-30分钟

关键是要真正做到专注，不被手机和其他干扰打断。
